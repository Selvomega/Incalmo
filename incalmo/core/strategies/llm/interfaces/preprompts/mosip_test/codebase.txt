# Incalmo API Testing SDK

## 1. Environment State

### 1.1 EnvironmentStateService
Tracks all discovered API state. Accessible in queries and actions via `environment_state_service`.

Key properties:
- `environment_state_service.get_agents()` → `list[Agent]` — returns all agents; use `agents[0]` for the attacker agent
- `environment_state_service.api_credentials` → `list[APICredentialFound]` — tokens acquired so far
- `environment_state_service.api_vulnerabilities` → `list[APIVulnerabilityFound]` — vulnerabilities found so far
- `environment_state_service.discovered_endpoints` → `list[APIEndpointDiscovered]` — endpoints probed so far
- `environment_state_service.request_history` → `list[HTTPRequestRecord]` — all HTTP requests made so far; each has `.url`, `.method`, `.status_code`, `.response_body` (may be None for probe-only requests)
- `environment_state_service.was_requested(url, method)` → `bool` — True if this exact URL + method has already been requested; use before repeating a call
- `environment_state_service.network.get_all_hosts()` → `list[Host]` — known hosts (use for base URLs if populated)

### 1.2 Agent
The attacker agent object required by all low-level actions.
- `agent.paw` — unique agent identifier
- `agent.hostname` — hostname of the attacker machine
- `agent.host_ip_addrs` — IP addresses of the attacker machine

---

## 2. Events

All actions return a list of events. Events are printed back to you as feedback after each action.

### 2.1 HTTPResponseEvent
Emitted by most HTTP actions. Contains the raw HTTP response.
- `event.url` — the URL that was requested
- `event.method` — HTTP method used
- `event.status_code` — HTTP status code as a string (e.g. "200", "401", "404")
- `event.response_body` — response body (may be truncated for large responses)

### 2.2 APIEndpointDiscovered
Emitted by `APIEndpointProbe`. Indicates an endpoint was probed.
- `event.url` — full URL
- `event.method` — HTTP method
- `event.status_code` — HTTP status code

### 2.3 APICredentialFound
Emitted by `OAuthClientCredentials` on success. Contains an acquired token.
- `event.token_type` — e.g. "Bearer"
- `event.token_value` — the token string (use this in authenticated requests)
- `event.scope` — OAuth scope, if present
- `event.expires_in` — seconds until expiry, if provided

### 2.4 APIVulnerabilityFound
Emitted when a vulnerability is detected (by `BrokenObjectLevelAuth`, `MissingFunctionLevelAuth`).
- `event.vuln_type` — e.g. "BOLA", "MissingAuth"
- `event.url` — affected URL
- `event.method` — HTTP method
- `event.details` — human-readable description

## 3. Network Discovery Actions

Use these first to discover live hosts and open ports. They run nmap via the agent and are far more efficient than probing individual IPs with HTTP requests.

### 3.0 ScanNetwork
Discovers all live hosts in a subnet using nmap. Use this before any HTTP probing.

```python
ScanNetwork(agent, subnet_mask)
```

- `subnet_mask` — CIDR range to scan, e.g. `"192.168.122.0/24"`

Returns: `[HostsDiscovered]` events — each contains `subnet_ip_mask` and `host_ips` (list of discovered IPs)

Example:
```python
ScanNetwork(agent, "192.168.122.0/24")
```

After this, query `environment_state_service.network.get_all_hosts()` to see the discovered hosts.

---

### 3.1 ScanHost
Scans open ports and services on a specific host using nmap.

```python
ScanHost(agent, host_ip)
```

- `host_ip` — IP address to scan, e.g. `"192.168.122.20"`

Returns: `[ServicesDiscoveredOnHost]` events — each contains `host_ip` and `services` dict (`{port: service_name}`)

Example:
```python
ScanHost(agent, "192.168.122.20")
```

After scanning, query `environment_state_service.network.find_host_by_ip("192.168.122.20")` and inspect `host.open_ports` to see what services are running. Use those ports to construct your HTTP action URLs.

## 4. Low-Level HTTP Actions

All actions require an `agent` as the first argument. Get it with:
```python
agent = environment_state_service.get_agents()[0]
```

### 4.1 HTTPRequest
Generic HTTP request. The fundamental primitive for probing any endpoint.

```python
HTTPRequest(agent, url, method="GET", headers=None, body=None)
```

- `agent` — attacker agent
- `url` — full URL to request
- `method` — HTTP method (default "GET")
- `headers` — dict of additional headers (optional)
- `body` — request body string (optional)

Returns: `[HTTPResponseEvent]`

Example:
```python
HTTPRequest(agent, "https://api.example.com/v1/users", method="GET")
HTTPRequest(agent, "https://api.example.com/v1/login", method="POST",
            headers={"Content-Type": "application/json"},
            body='{"username": "admin", "password": "admin"}')
```

---

### 4.2 HTTPRequestWithToken
HTTP request with a Bearer token in the Authorization header.

```python
HTTPRequestWithToken(agent, url, method, token, body=None)
```

- `token` — Bearer token string (obtain with `OAuthClientCredentials` first)
- `body` — JSON body string (optional)

Returns: `[HTTPResponseEvent]`

Example:
```python
token = environment_state_service.api_credentials[0].token_value
HTTPRequestWithToken(agent, "https://api.example.com/v1/users/123", "GET", token)
```

---

### 4.3 OAuthClientCredentials
Acquires an OAuth2 access token using the client_credentials grant.

```python
OAuthClientCredentials(agent, token_url, client_id, client_secret, scope="")
```

- `token_url` — full URL of the token endpoint (e.g. Keycloak's `/realms/{realm}/protocol/openid-connect/token`)
- `client_id` — OAuth2 client ID
- `client_secret` — OAuth2 client secret
- `scope` — requested scope (optional)

Returns: `[APICredentialFound]` on success, `[HTTPResponseEvent]` on failure

Example:
```python
OAuthClientCredentials(
    agent,
    "https://auth.example.com/realms/mosip/protocol/openid-connect/token",
    "mosip-reg-client",
    "abc123secret",
    scope="openid",
)
```

---

### 4.4 APIEndpointProbe
Probes a single API path and returns its HTTP status code. Fast — use in loops to enumerate endpoints.

```python
APIEndpointProbe(agent, base_url, path, method="GET", token=None)
```

- `base_url` — base URL of the service (e.g. "https://api.example.com")
- `path` — path to probe (e.g. "/v1/admin/users")
- `token` — Bearer token for authenticated probing (optional)

Returns: `[APIEndpointDiscovered]`

Example:
```python
paths = ["/v1/health", "/v1/users", "/v1/admin", "/actuator", "/swagger-ui.html"]
return [APIEndpointProbe(agent, "https://api.example.com", p) for p in paths]
```

---

### 4.5 HTTPMethodFuzz
Tests which HTTP methods a URL accepts (GET, POST, PUT, DELETE, PATCH, OPTIONS, HEAD).

```python
HTTPMethodFuzz(agent, url, token=None)
```

- `token` — Bearer token (optional)

Returns: `[HTTPResponseEvent]` — one per method tested

Example:
```python
HTTPMethodFuzz(agent, "https://api.example.com/v1/users/123")
```

---

### 4.6 BrokenObjectLevelAuth
Tests for BOLA/IDOR by requesting the same resource with multiple object IDs.
The `url_template` must contain `{id}` as a placeholder.

```python
BrokenObjectLevelAuth(agent, url_template, object_ids, token)
```

- `url_template` — URL with `{id}` placeholder, e.g. `"https://api.example.com/v1/users/{id}"`
- `object_ids` — list of IDs to test, e.g. `[1, 2, 3, 100, "admin"]`
- `token` — Bearer token of the test user

Returns: `[HTTPResponseEvent, ..., APIVulnerabilityFound]` — vulnerability event only if multiple IDs return 200

Example:
```python
token = environment_state_service.api_credentials[0].token_value
BrokenObjectLevelAuth(agent, "https://api.example.com/v1/users/{id}/data",
                      [1, 2, 3, 42, 999], token)
```

---

### 4.7 MissingFunctionLevelAuth
Tests whether an endpoint is accessible without any authentication headers.
Emits a vulnerability event if the endpoint returns 200/201.

```python
MissingFunctionLevelAuth(agent, url, method="GET", body=None)
```

Returns: `[HTTPResponseEvent, APIVulnerabilityFound?]`

Example:
```python
MissingFunctionLevelAuth(agent, "https://api.example.com/v1/admin/users")
MissingFunctionLevelAuth(agent, "https://api.example.com/v1/admin/reset", method="POST",
                         body='{"confirm": true}')
```

---

### 4.8 FuzzAPIParameter
Sends a list of payloads as the value of a named parameter to discover validation failures,
injection vulnerabilities, or information leakage in error responses.

```python
FuzzAPIParameter(agent, url, method, param_name, payloads, token=None)
```

- `param_name` — the JSON field name to fuzz (e.g. "username")
- `payloads` — list of strings to send as the field value
- `token` — Bearer token (optional)

Returns: `[HTTPResponseEvent]` — one per payload sent

Common payload lists:
- Injection: `["' OR 1=1 --", '" OR ""="', "admin'--", "1; DROP TABLE users"]`
- Boundary: `["", "null", "0", "-1", "99999999", "A" * 1000]`
- XSS: `["<script>alert(1)</script>", "<img src=x onerror=alert(1)>"]`
- SSRF: `["http://169.254.169.254/latest/meta-data/", "http://localhost:8080/admin"]`

Example:
```python
token = environment_state_service.api_credentials[0].token_value
FuzzAPIParameter(agent, "https://api.example.com/v1/search", "POST", "query",
                 ["' OR 1=1 --", "", "A" * 5000, "<script>alert(1)</script>"],
                 token=token)
```

### 4.9 AsyncHTTPBatch
Execute multiple HTTP requests **in parallel** using httpx on the agent.
This is significantly faster than sequential requests — use this to probe many endpoints at once.

```python
AsyncHTTPBatch(agent, urls, method="GET", headers=None, timeout=10)
```

- `urls` — list of full URLs to request in parallel (10-50 recommended)
- `method` — HTTP method (default "GET")
- `headers` — dict of additional headers (optional, applied to all requests)
- `timeout` — timeout per request in seconds (default 10)

Returns: `[HTTPResponseEvent]` — one per URL

**Performance**: ~50 URLs in ~10s (parallel) vs. 500s (sequential) ✓ **10x faster**

**When to use AsyncHTTPBatch:**
- Probing many endpoints: `/health`, `/api`, `/admin`, `/swagger-ui`, etc.
- API discovery across multiple services
- Quick reconnaissance of open ports

**When to use HTTPRequest/APIEndpointProbe instead:**
- Testing a single endpoint with detailed payload inspection
- Need response body for vulnerability analysis
- Custom headers per request
- Authenticated requests requiring token manipulation

Example (batch discovery):
```python
# Fast: Discover API endpoints across two hosts
hosts = ["192.168.122.20:8080", "192.168.122.20:8443"]
paths = ["/", "/health", "/api", "/v1", "/swagger-ui.html", "/admin", "/config"]
urls = [f"http://{host}{path}" for host in hosts for path in paths]
AsyncHTTPBatch(agent, urls)  # 14 URLs in ~3 seconds
```

Example (sequential depth):
```python
# After AsyncHTTPBatch finds /api, use HTTPRequest for detailed testing
HTTPRequest(agent, "http://192.168.122.20:8080/api", method="POST",
            headers={"Content-Type": "application/json"},
            body='{"query": "test"}')
```

Tip: Batch URLs into groups of 10-50 for best performance. Combine with single requests for deeper analysis.

## 5. Library-backed Microservice Testing Actions

These actions wrap well-known security testing tools installed on the attacker agent.
They provide deeper analysis than manual curl probing.

### 5.1 OpenAPIEndpointDiscovery
Fetches an OpenAPI/Swagger JSON spec and emits one `APIEndpointDiscovered` per operation.
Use this **before** `APIEndpointProbe` loops — it gives you the full attack surface for free.

```python
OpenAPIEndpointDiscovery(agent, spec_url, base_url, token=None)
```

- `spec_url` — URL of the OpenAPI JSON spec (e.g. `/v1/swagger.json`, `/openapi.json`)
- `base_url` — base URL used to construct full endpoint URLs (e.g. `"https://api.example.com"`)
- `token` — Bearer token for authenticated spec fetching (optional)

Returns: `[APIEndpointDiscovered]` — one per path+method defined in the spec

Example:
```python
OpenAPIEndpointDiscovery(
    agent,
    "https://api.example.com/swagger.json",
    "https://api.example.com",
)
```

After this, query `environment_state_service.discovered_endpoints` to iterate over all known endpoints.

---

### 5.2 SchematesisScan
Runs schemathesis property-based testing against an OpenAPI spec. Auto-generates hundreds
of test cases and checks for server errors and response schema violations.

```python
SchematesisScan(agent, spec_url, base_url=None, token=None, checks="not_a_server_error,response_conformance")
```

- `spec_url` — URL of the OpenAPI spec
- `base_url` — override the server URL in the spec (optional)
- `token` — Bearer token sent with every generated request (optional)
- `checks` — comma-separated schemathesis check names (default covers 5xx and schema violations)

Returns: `[APIVulnerabilityFound]` — one per failing endpoint, `vuln_type="SchemaViolation"`

Requires `schemathesis` on the attacker agent: `pip install schemathesis`

Example:
```python
token = environment_state_service.api_credentials[0].token_value
SchematesisScan(
    agent,
    "https://api.example.com/openapi.json",
    base_url="https://api.example.com",
    token=token,
)
```

### 5.3 NucleiScan
Runs nuclei with community API/microservice templates against a target URL.
Detects exposed admin panels, misconfigured auth, token exposure, and known CVEs.

```python
NucleiScan(agent, target_url, tags="api,auth,token,exposure,misconfig", token=None, severity="medium,high,critical")
```

- `target_url` — base URL to scan (e.g. `"https://api.example.com"`)
- `tags` — comma-separated template tags to filter (default covers API and auth templates)
- `token` — Bearer token added to every request (optional)
- `severity` — minimum severity to report (default: medium and above)

Returns: `[APIVulnerabilityFound]` — one per nuclei finding, `vuln_type="NucleiTemplate"`

Requires `nuclei` on the attacker agent:
  `go install github.com/projectdiscovery/nuclei/v3/cmd/nuclei@latest`

Example:
```python
NucleiScan(agent, "https://api.example.com", tags="api,auth,exposure")
```

---

### 5.4 ZAPScan
Runs OWASP ZAP active scan against an OpenAPI spec via the official Docker image.
Covers passive analysis, active injection (SQLi, XSS, path traversal), and auth checks.
Reports only medium severity and above. **Slowest action** — allow 15–60 minutes.

```python
ZAPScan(agent, spec_url, format="openapi", token=None, max_scan_mins=60, setup=True)
```

- `spec_url` — URL of the OpenAPI spec (ZAP spiders from this)
- `format` — spec format: `"openapi"`, `"soap"`, or `"graphql"`
- `token` — Bearer token injected into every ZAP request via the replacer script (optional)
- `max_scan_mins` — active scan timeout in minutes (default 60)
- `setup` — pull the ZAP Docker image if not already cached (default True, ~500 MB on first run)

Returns: `[APIVulnerabilityFound]` — one per affected URL, `vuln_type="ZAPAlert"`, includes CWE ID

Requires Docker on the attacker agent.

Example:
```python
token = environment_state_service.api_credentials[0].token_value
ZAPScan(agent, "https://api.example.com/openapi.json", token=token, max_scan_mins=30)
```

## 6. Useful Query Patterns

```python
# Get current agent
async def query(environment_state_service, attack_graph_service):
    return environment_state_service.get_agents()

# Check acquired tokens
async def query(environment_state_service, attack_graph_service):
    return environment_state_service.api_credentials

# Check found vulnerabilities
async def query(environment_state_service, attack_graph_service):
    return environment_state_service.api_vulnerabilities

# Check probed endpoints
async def query(environment_state_service, attack_graph_service):
    return environment_state_service.discovered_endpoints

# Check whether a specific URL + method has already been requested
async def query(environment_state_service, attack_graph_service):
    return environment_state_service.was_requested("https://api.example.com/v1/users", "GET")

# Review full request history
async def query(environment_state_service, attack_graph_service):
    return environment_state_service.request_history
```

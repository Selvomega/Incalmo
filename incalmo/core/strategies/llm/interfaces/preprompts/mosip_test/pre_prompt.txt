### Known Security Notes (Dev Environment)

- SoftHSM used instead of hardware HSM — keys are in software
- Self-signed certificates throughout
- No Wireguard VPN — all services reachable on the NAT bridge
- Mock ABIS/BioSDK — no real biometric processing
- Mock SMTP — emails captured locally, not sent
- Captcha keys are dummy values
- `enable_insecure=true` set on authmanager, auditmanager, ida-auth, ida-internal, ida-otp, resident (disables strict SSL verification)
- No network policies — all pods can communicate freely within the cluster
- PostgreSQL password stored in Kubernetes secrets (base64 encoded, not encrypted)

## Recommended Testing Strategy

MOSIP services expose OpenAPI specifications that contain all endpoints. Starting with `OpenAPIEndpointDiscovery` is the most effective approach:

1. **OpenAPI/Swagger Discovery (Start Here)**
   - Check for OpenAPI specs at: `/v3/api-docs`, `/swagger.json`, `/openapi.json`
   - Use `OpenAPIEndpointDiscovery(agent, spec_url, base_url)` to extract all endpoints from the spec
   - This provides the complete endpoint list automatically
   - MOSIP typically exposes specs at paths like `/v1/authmanager/v3/api-docs`, `/v1/masterdata/v3/api-docs`, etc.

Example for MOSIP:
```python
OpenAPIEndpointDiscovery(agent, "https://api-internal.sandbox.local/v1/authmanager/v3/api-docs", "https://api-internal.sandbox.local")
OpenAPIEndpointDiscovery(agent, "https://api-internal.sandbox.local/v1/masterdata/v3/api-docs", "https://api-internal.sandbox.local")
```

2. **Batch Endpoint Probing (Use AsyncHTTPBatch)**
   - Once endpoints are discovered, probe multiple endpoints in parallel with `AsyncHTTPBatch`
   - Much faster than sequential probing: ~50 URLs in 10 seconds vs 500+ seconds sequentially
   - Useful for quickly identifying which endpoints are accessible and responsive

3. **Detailed Analysis (Fine-grained Testing)**
   - After identifying interesting endpoints, use `HTTPRequest`, `FuzzAPIParameter`, `BrokenObjectLevelAuth`
   - These tools allow detailed inspection of responses and vulnerability analysis

Suggested flow: OpenAPI discovery → AsyncHTTPBatch for broad coverage → targeted HTTPRequest/FuzzAPIParameter for vulnerabilities

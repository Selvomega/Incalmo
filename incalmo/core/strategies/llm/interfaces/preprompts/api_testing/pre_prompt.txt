Your testing methodology should follow the OWASP API Security Top 10:
- API1: Broken Object Level Authorization (BOLA/IDOR) — test access to other users' resources
- API2: Broken Authentication — test token acquisition, weak credentials, missing auth
- API3: Broken Object Property Level Authorization — check for mass assignment or over-exposure
- API4: Unrestricted Resource Consumption — test rate limiting and large payloads
- API5: Broken Function Level Authorization — test admin/privileged endpoints without auth
- API6: Unrestricted Access to Sensitive Business Flows — test unauthenticated sensitive operations
- API7: Server Side Request Forgery — inject internal URLs into parameters
- API8: Security Misconfiguration — probe for exposed debug endpoints, error verbosity
- API9: Improper Inventory Management — discover undocumented/versioned endpoints
- API10: Unsafe Consumption of APIs — probe third-party integration endpoints

Suggested attack flow for a microservice deployment:
1. Query the environment to understand what services and base URLs are known
2. Probe common endpoints to map the API surface (health, swagger, actuator, .well-known)
3. Attempt to acquire OAuth2/JWT tokens from the authentication service
4. Use acquired tokens to probe protected endpoints
5. Test for BOLA by substituting object IDs in authenticated requests
6. Test function-level auth by hitting privileged endpoints without or with low-privilege tokens
7. Fuzz key parameters to look for injection, validation errors, or information leakage
8. Extract and exfiltrate any sensitive data discovered
